# 유지보수 사례 검색기 개발기능 추가 후보

작성일: 2026-06-11

## 확인한 현재 상태

- 기존 개발물은 `app.py`, `README.md`, `requirements.txt`, `build_exe.ps1` 중심의 단일 GUI 프로그램이다.
- 기능 구조는 유지보수 엑셀 파일 수집, SQLite/joblib 캐시 저장, BM25 + TF-IDF 유사 사례 검색, 연도/부서/사용자/APC/PC filter/UTMP 필터, tkinter 결과 화면으로 구성되어 있다.
- 현재 추출 로직은 모든 `.xlsx`의 첫 번째 시트를 읽고 5행부터 사례로 변환한다.
- 2025.01~2026.04 데이터 16개 파일을 기준으로 확인하면, 현재 앱과 유사한 기준으로는 4,420행이 잡힐 수 있지만 장애내용 또는 조치내용이 있는 실제 유효 사례는 2,639건이다. 즉, 빈 템플릿 행 1,781건이 인덱스에 섞일 가능성이 있다.
- 문법 확인은 새 conda 환경 `maintenance_search_analysis`에서 `python -m py_compile app.py`로 통과했다.

## 추천 작업 순서

1. 유효 사례 추출 기준 개선과 인덱스 구축 리포트부터 처리한다.
2. 월/기간 필터, 부서/사용자 자동완성, 결과 내보내기를 붙여 실무 검색기로 완성도를 올린다.
3. 장애유형 자동 분류와 월별/부서별 대시보드를 추가해 분석 기능으로 확장한다.
4. 반복/유사 사례 그룹과 표준 조치 추천을 붙여 보고서와 매뉴얼 작성에 활용한다.
5. 마지막으로 테스트, 로깅, 캐시 자동 갱신, 배포 안정화를 정리한다.
