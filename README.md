# 유지보수 사례 검색기

유지보수 내역서 엑셀 파일을 읽어서 장애내용, 조치내용, 부서, 사용자, 날짜 등을 인덱싱하고, 다음 순서로 유사 사례를 찾는 GUI 프로그램입니다.

일반 사용자에게 배포할 때는 `사용자_간단_매뉴얼.md`를 함께 전달하세요. 제출·배포 구성은 `배포_제출_안내.md`를 참고하세요.

1. 키워드 검색: BM25
2. 부분어/유사 표현 검색: 문자 2~5-gram TF-IDF 기반 코사인 유사도
3. 조건 필터: 연도, 부서, 사용자, APC / PC filter / UTMP

BM25와 벡터 검색은 필터를 통과한 전체 사례에 함께 적용됩니다.

## 준비 사항

- Windows
- Python 3
- 검색할 유지보수 내역서 Excel 파일(`.xlsx`)

유지보수 양식으로 판단되는 시트만 읽습니다. 장애내용/조치내용 같은 주요 항목이 없는 표지, 요약, 분석용 시트는 인덱스에서 제외될 수 있습니다.


## 소스에서 실행

저장소를 내려받은 폴더에서 다음 명령을 실행합니다.

```powershell
py -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
.\.venv\Scripts\python.exe app.py
```

Windows에서 `app.py`를 더블클릭하면 PC의 파일 연결 설정에 따라 Jupyter Notebook 같은 다른 프로그램이 열릴 수 있습니다. 이 경우 `run_app.bat`를 실행하거나, 아래 방식으로 생성한 `.exe` 파일을 사용하는 것이 안전합니다.

## 실행 파일 생성

다음 명령은 `.venv`가 없으면 자동으로 만들고 필요한 패키지를 설치한 뒤 실행 파일을 생성합니다.

```powershell
powershell -ExecutionPolicy Bypass -File .\build_exe.ps1
```

PowerShell 명령이 익숙하지 않으면 `build_exe.bat`를 더블클릭해도 됩니다.

생성된 파일:

- `dist\유지보수사례검색기.exe`

다른 PC에는 이 실행 파일만 전달해도 프로그램을 실행할 수 있습니다. Python 설치는 필요하지 않지만, 검색할 Excel 파일은 별도로 준비해야 합니다.


## 처음 사용하는 방법

1. 프로그램을 실행합니다.
2. `찾기`를 눌러 유지보수 Excel 파일들이 들어 있는 최상위 폴더를 선택합니다.
3. `인덱스 구축`을 누릅니다.
4. 구축이 완료되면 검색어와 필터를 입력해 검색합니다.

인덱스는 선택한 데이터 폴더 아래의 `.maintenance_search_cache` 폴더에 저장됩니다. 같은 폴더를 다시 사용할 때는 저장된 인덱스를 자동으로 불러옵니다.

Excel 파일이 추가, 수정, 삭제되면 프로그램 실행 중에도 상단에 데이터 변경 안내가 표시됩니다. 이 경우 `인덱스 구축`을 눌러 새로 구축하면 최신 내용이 검색에 반영됩니다.

`인덱스 리포트 보기`에서는 읽은 파일, 제외 파일, 제외 행, 누락 항목 등 인덱스 구축 결과를 확인할 수 있습니다.

## 내역 추가·수정

- `새 내역 추가`: 대상 연도/월과 일자를 선택한 뒤 순번, 부서, 사용자, 장애내용, 조치내용을 입력해 원본 Excel 파일에 새 행을 추가합니다. 순번은 자동으로 제안됩니다.
- `원본 열기`: 선택한 사례의 원본 Excel 파일을 엽니다. Windows에서 Excel 연동이 가능하면 해당 행 위치로 이동합니다.
- `선택 항목 수정`: 검색 결과에서 선택한 사례의 순번, 날짜, 부서, 사용자, 장애내용, 조치내용, APC/PC filter/UTMP 값을 수정합니다.

새 내역은 선택한 연도/월의 기존 유지보수 Excel 파일 마지막 데이터 행 아래에 추가됩니다. 프로그램은 바로 윗행의 서식을 복사해 기존 양식과 같은 형태로 저장하며, 없는 월의 새 Excel 파일은 자동 생성하지 않습니다.

추가 또는 수정 후에는 안내에 따라 `인덱스 구축`을 다시 실행해야 변경 내용이 검색 결과에 반영됩니다.
