# 유지보수 사례 검색기 배포·제출 안내

## 1. 제출 권장 구성

주무관님께 제출할 때는 아래 파일을 중심으로 전달하는 것을 권장합니다.

- `유지보수사례검색기.exe`
- `사용자_간단_매뉴얼.md`
- `제출_체크리스트.md`
- `README.md`
- `의왕시 유지보수 내역 분석 및 사례검색기 활용 보고서_가독성개선본.txt`
- `visualizations/` 폴더의 차트 이미지

내부 작업 메모인 `TODO.md`, 개발 후보 문서, 분석 초안 문서는 제출용에서 제외하는 것이 좋습니다.

## 2. 실행 파일 생성

Windows PowerShell에서 `maintenance_search` 폴더로 이동한 뒤 실행합니다.

```powershell
powershell -ExecutionPolicy Bypass -File .\build_exe.ps1
```

또는 `build_exe.bat`를 더블클릭합니다.

생성 파일:

- `dist\유지보수사례검색기.exe`

일반 사용자에게는 이 `.exe` 파일을 전달하는 방식이 가장 편합니다.

## 3. 배포 폴더 예시

```text
유지보수사례검색기_배포/
  유지보수사례검색기.exe
  사용자_간단_매뉴얼.md
  visualizations/
```

유지보수 원본 엑셀 파일은 별도 데이터 폴더로 관리하는 것이 좋습니다.

## 4. 사용 PC 안내

- Windows PC에서 사용합니다.
- `.exe` 실행 방식은 Python 설치가 필요하지 않습니다.
- `app.py`를 직접 더블클릭하지 않습니다. PC 설정에 따라 Jupyter Notebook 등 다른 프로그램이 열릴 수 있습니다.

## 5. 제출 전 확인사항

- 실행 파일이 정상 실행되는지 확인
- `찾기`로 데이터 폴더 선택 가능 여부 확인
- `인덱스 구축` 완료 여부 확인
- 검색어 입력 후 결과 표시 여부 확인
- `전체 보기` 동작 확인
- `인덱스 리포트 보기`에서 유효 사례와 제외 행 확인
- `새 내역 추가`, `원본 열기`, `선택 항목 수정` 동작 확인
- 새 내역 추가 시 순번 자동 입력 및 수정 가능 여부 확인
- 새 내역 추가 시 선택한 연도/월의 저장 파일 표시 여부 확인
- 검색 결과 엑셀 저장 가능 여부 확인

## 6. 데이터 관리 주의사항

- 검색 결과 파일, 보고서 파일, 분석 산출물은 원본 유지보수 데이터 폴더에 섞지 않는 것이 좋습니다.
- 프로그램은 검색결과, 보고서, 분석, result, output, backup, bak 등이 포함된 엑셀 파일을 인덱싱 대상에서 제외합니다.
- 유지보수 양식의 주요 항목이 없는 표지, 요약, 분석용 시트는 인덱싱 대상에서 제외될 수 있습니다.
- 엑셀 내용이 바뀌면 프로그램 실행 중에도 상단의 데이터 변경 안내를 확인한 뒤 `인덱스 구축`을 눌러 새로 구축합니다.
- 프로그램에서 내역을 추가하거나 수정한 경우에도 `인덱스 구축`을 다시 실행해야 최신 검색 결과에 반영됩니다.
