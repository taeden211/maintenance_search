app.py의 유지보수 사례 검색기 코드를 아래 기준으로 개선해줘.

작업 대상:

* /Users/pilt/project-collection/workbench/현장실습/maintenance_search/app.py

목표:
기존 검색 구조는 유지하되, 유효 사례 추출 기준, 검색 점수 정규화, 성능, 필터 안정성을 개선한다. 기존 UI와 기능은 깨지지 않게 최소 변경으로 수정한다.

수정 요구사항:

1. 유효 사례 추출 기준 개선

현재 _collect_cases()에서 5행부터 읽으면서 날짜, 연번, 부서, 사용자만 있어도 사례로 들어갈 수 있다.

문제:

* 장애내용과 조치내용이 둘 다 비어 있는 빈 템플릿 행도 인덱스에 포함될 수 있다.
* 기존 후보 문서 기준 빈 템플릿 행 후보가 약 1,781건이라 검색 품질에 영향이 크다.

수정:

* issue_text와 action_text가 둘 다 비어 있으면 해당 행은 사례로 저장하지 않는다.
* 날짜, 연번, 부서, 사용자만 있는 행은 제외한다.
* 단, 장애내용 또는 조치내용 중 하나라도 있으면 유효 사례로 포함한다.

예상 코드 방향:

if not issue_text and not action_text:
    continue

2. 검색 결과 0건 버그 가능성 수정

현재 _normalize_scores()에서 max_value와 min_value가 같으면 전부 0점으로 만든다.

문제:

* 필터를 좁혀 후보가 1건만 남은 경우
* 모든 후보 점수가 같은 경우
* 실제 매칭이 있는데도 정규화 결과가 전부 0이 되어 positive_indices가 없어지고 검색 결과가 0건이 될 수 있다.

수정:

* 모든 점수가 같더라도 양수 점수라면 1.0으로 정규화한다.
* 모든 점수가 0 이하이면 0으로 유지한다.

예상 코드 방향:

if np.isclose(max_value, min_value):
    if max_value > 0:
        return np.ones_like(values, dtype=np.float32)
    return np.zeros_like(values, dtype=np.float32)

3. BM25 성능 개선

현재 BM25Index.score()는 검색할 때마다 각 문서의 모든 토큰을 순회하며 term frequency를 계산한다.

문제:

* 데이터가 많아질수록 검색 속도가 느려질 수 있다.

수정:

* BM25Index.__init__()에서 문서별 Counter를 미리 만들어 저장한다.
* score()에서는 self.term_freqs[idx].get(term, 0) 방식으로 빈도를 조회한다.
* 클래스 필드가 변경되므로 SEARCH_INDEX_VERSION을 올린다.

예상 코드 방향:

from collections import Counter
self.term_freqs = [Counter(doc) for doc in tokenized_docs]

4. 연도 필터 안정성 개선

현재 연도 필터는 record.year가 0이면 필터를 통과할 수 있다.

문제:

수정:

* 연도 필터가 설정되어 있는데 record.year가 0이면 제외한다.

예상 코드 방향:

if filters.year_from is not None:

if filters.year_to is not None:
    if not record.year or record.year > filters.year_to:

5. APC / PC filter / UTMP 체크값 판정 개선

현재는 값이 정확히 "O"일 때만 체크된 것으로 본다.

문제:

* 엑셀에 o, O , ○, Y, 예, 1 등으로 들어가면 필터에서 누락될 수 있다.

수정:

* 체크 판정 함수를 추가한다.
* 다음 값들은 체크된 것으로 인정한다: O, Y, YES, TRUE, 1, ○, 예
* 대소문자와 앞뒤 공백은 무시한다.

예상 코드 방향:

def is_checked(value: str) -> bool:
    return value.strip().upper() in {"O", "Y", "YES", "TRUE", "1", "○", "예"}

현재 load_workbook() 실패 시 전체 인덱스 구축이 중단될 수 있다.

수정:

try:

유지보수 원본 데이터가 아닌 파일은 인덱싱 대상에서 제외한다.

예상 대상:

* 백업 파일
* 임시 파일
* 사용자가 생성한 산출물

예상 방향:

EXCLUDE_FILE_KEYWORDS = [
    "검색결과",
    "보고서",
    "분석",
    "result",
    "output",
    "backup",
    "bak",
]
def should_skip_excel_file(path: Path) -> bool:
    if path.name.startswith("~$"):
        return True
    lowered = path.name.lower()
    return any(
        keyword.lower() in lowered
        for keyword in EXCLUDE_FILE_KEYWORDS
    )
