# 유지보수 사례 검색기 사용자 간단 매뉴얼

## 1. 프로그램 용도

월별 유지보수 내역서의 장애내용, 조치내용, 부서, 사용자, 날짜 등을 검색하여 과거 유사 사례를 빠르게 확인하는 프로그램입니다.

## 2. 실행 방법

권장 실행 파일:

- `유지보수사례검색기.exe`

개발/점검용 실행 파일:

- `run_app.bat`

주의: `app.py`를 더블클릭하면 PC 설정에 따라 Jupyter Notebook 등 다른 프로그램이 열릴 수 있습니다. 일반 사용자는 `.exe` 파일을 실행하는 방식이 가장 편합니다.

## 3. 처음 사용할 때

1. 프로그램을 실행합니다.
2. `찾기`를 눌러 유지보수 내역서가 들어 있는 최상위 폴더를 선택합니다.
3. `인덱스 구축`을 누릅니다.
4. 완료 안내가 뜨면 검색어와 필터를 입력해 사용합니다.

인덱스는 선택한 데이터 폴더 아래 `.maintenance_search_cache` 폴더에 저장됩니다.

Excel 파일이 추가, 수정, 삭제되면 프로그램 실행 중에도 상단에 데이터 변경 안내가 표시됩니다. 안내가 뜨면 `인덱스 구축`을 눌러 새로 구축합니다.

## 4. 검색 방법

1. `검색어` 입력칸에 증상, 조치, 부서, 사용자, 날짜 등 확인할 키워드를 입력합니다.
2. 필요한 경우 연도, 월, 부서, 사용자, APC, PC filter, UTMP 조건을 선택합니다.
3. `검색`을 누릅니다.
4. 목록에서 사례를 선택하면 아래 상세 영역에서 전체 장애내용과 조치내용을 확인할 수 있습니다.

예시 검색어:

- 프린터 인쇄 오류
- 온나라 공문 열람
- 한글 열람 에러
- 스캔 에러
- 비밀번호 로그인
- 회계과
- 9월 26일

## 5. 원본 확인 및 내역 수정

- `원본 열기`: 선택한 사례의 원본 Excel 파일을 엽니다.
- `선택 항목 수정`: 선택한 사례의 순번, 날짜, 부서, 사용자, 장애내용, 조치내용 등을 수정합니다.
- `새 내역 추가`: 대상 연도/월과 일자를 선택해 월별 Excel 파일에 추가합니다. 순번은 자동으로 채워지며 필요하면 수정할 수 있습니다.

새 내역은 선택한 연도/월의 기존 Excel 파일 마지막 행 아래에 추가되며, 기존 행 서식을 따라 저장됩니다.

추가 또는 수정 후 안내가 뜨면 `인덱스 구축`을 눌러 최신 내용으로 다시 반영합니다.

주의: 원본 Excel 파일이 이미 열려 있으면 저장이 실패할 수 있습니다. 이 경우 Excel 파일을 닫고 다시 시도합니다.

## 6. 전체 보기

`전체 보기`를 누르면 현재 필터 조건에 해당하는 전체 사례를 볼 수 있습니다.

검색어 없이 특정 월, 부서, 사용자 조건만 확인할 때 사용합니다.

## 7. 인덱스 리포트 보기

`인덱스 리포트 보기`에서는 다음 내용을 확인할 수 있습니다.

- 읽은 파일 수
- 제외 파일 수
- 유효 사례 수
- 제외 행 수
- 부서/사용자/APC/PC filter/UTMP 누락 건수

`데이터 품질` 탭에서 `상세 행 위치 표시`를 켜면 누락 또는 제외된 행의 파일명, 시트명, 행번호를 확인할 수 있습니다.

## 8. 엑셀 저장

검색 결과 또는 전체 사례 목록을 확인한 뒤 `검색 결과 엑셀 저장`을 누르면 현재 화면의 결과를 `.xlsx` 파일로 저장할 수 있습니다.

## 9. 사용 시 주의사항

- 유지보수 내역서 파일이 추가되거나 수정되면 인덱스를 새로 구축해야 합니다.
- 장애내용/조치내용 등 유지보수 양식의 주요 항목을 가진 Excel 시트만 인식됩니다. 표지, 요약, 분석용 시트처럼 유지보수 양식이 아닌 시트는 제외될 수 있습니다.
- 검색 결과는 과거 사례 참고용입니다. 실제 조치 전에는 장비 상태, 프로그램 버전, 보안 설정을 확인해야 합니다.
- 개인정보가 포함될 수 있으므로 검색 결과 엑셀 파일은 필요한 범위에서만 공유합니다.
